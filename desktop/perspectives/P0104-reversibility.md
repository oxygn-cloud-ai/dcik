# Perspective: Reversibility (Type 1 / Type 2 Decisions)

**ID:** P0104
**Domain:** Decision-making, strategy, organisational design
**Source:** Jeff Bezos (Type 1/Type 2 decisions)
**Invoke when:** The assessment treats all decisions as equally consequential or applies the same decision-making process regardless of reversibility.

## Lens

Bezos distinguished Type 1 decisions (irreversible, high-stakes — take time, get it right) from Type 2 decisions (reversible, low-stakes — make them fast, correct if wrong). Most organisations treat Type 2 decisions as Type 1, creating decision bottlenecks. Most failures come from treating Type 1 decisions as Type 2 (rushing irreversible commitments).

1. **Decision classification:** For each decision: Type 1 (irreversible or prohibitively expensive to reverse) or Type 2 (reversible at acceptable cost)? Be honest — most decisions are Type 2 but are treated as Type 1.
2. **Reversibility cost:** What does it cost to reverse? Money, time, reputation, relationships, opportunity. If the reversal cost is low, the decision should be fast. Speed of learning > quality of initial decision.
3. **Type 2 decision speed:** Are Type 2 decisions being escalated, delayed, over-analysed? The cost of delay for a reversible decision often exceeds the cost of being wrong. Make Type 2 decisions at the lowest possible level.
4. **Type 1 decision depth:** Are Type 1 decisions receiving proportionate analysis? One-way-door decisions deserve deep analysis, external input, and deliberate process. Most assessments apply the same process to all decisions.
5. **Decisional regret minimisation:** Bezos' framework: project yourself to age 80. Will you regret not having made this decision? Regret minimisation frames decisions in terms of life regret, not quarterly earnings.
6. **Asymmetric reversibility:** Some decisions are reversible but the window is limited. A decision that can be reversed today but not next month is currently Type 2 but becoming Type 1.

## Default adversarial stance

Assume Type 2 decisions are being treated as Type 1 (over-analysed, delayed). Assume Type 1 decisions are being treated as Type 2 (insufficiently analysed). Classify first, then decide.
