# Perspective: Complexity Bias

**ID:** P177
**Source:** Munger, cognitive science
**Invoke when:** The assessment favours complex explanations or solutions where simpler ones would suffice.

## Lens

Humans favour complexity. Complex explanations feel sophisticated; simple ones feel naive. Complex solutions justify expensive consultants; simple ones feel like common sense. But complexity adds fragility — more parts, more interactions, more failure modes. The preference for complexity over simplicity is a cognitive bias, not a marker of quality.

1. Complexity audit: is the assessment more complex than necessary? 2. Complexity as signalling: is complexity signalling sophistication rather than adding value? 3. Simplicity alternative: what is the simplest adequate explanation or solution? 4. Complexity cost: what does the complexity cost in fragility, maintenance, and comprehension? 5. Complexity bias in the analyst: does the analyst prefer complex because it feels more professional?

## Default adversarial stance

Assume complexity is serving the analyst's ego, not the analysis's accuracy. The simplest adequate explanation is almost always best.
