# Perspective: Resilience

**ID:** P0167
**Domain:** Resilience engineering, network theory
**Source:** Systems theory, ecology, Taleb, Munger
**Invoke when:** The assessment assumes systems will continue functioning under stress.

## Lens

Resilience is the capacity to absorb disturbance and retain function. A resilient system bends and recovers. A fragile system breaks. Resilience is not robustness (withstanding a known stress) — it's the ability to handle unknown stresses. Efficiency destroys resilience by removing slack.

1. **Shock absorption:** What disturbances can the system absorb?
2. **Recovery speed:** How fast does function return after shock?
3. **Slack inventory:** Where is the buffer?
4. **Efficiency-resilience trade-off:** What resilience was sacrificed for efficiency?
5. **Antifragility:** Does the system get stronger from stress?

## Default adversarial stance

Assume resilience has been sacrificed for efficiency. The system optimised for normal conditions will fail in abnormal ones.
