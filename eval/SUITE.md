# Eval Topic Suite & Held-Out Methodology

Goal: **demonstrate** that a full DCIK run is powerfully useful — beats both a plain pass and the
"second model + disconfirming search" lever — *credibly*, i.e. in a way a skeptic accepts.

## Why topic class matters

The first PoC (`results/poc-2026-07-06/`) tested `sqlite-vs-postgres` — a technical lookup a
strong model half-knows, exactly where the LITE lever suffices. FULL tied/lost there. That is a
fair result for that class. DCIK's value proposition lives in a different class:

| Class | LITE likely enough? | DCIK's edge |
|---|---|---|
| Technical lookup (sqlite-vs-postgres) | Yes | small — model + search covers it |
| **High-stakes / ambiguous / adversarial / multi-stakeholder** (acquisition, negotiation, irreversible bet, contested policy) | **No** | **large — premise-failure, counterparty incentives, information asymmetry, decision-maker bias are systematically forced by the lens battery, not by one pass** |

The suite is therefore **weighted to the second class**, and results are reported **per class** —
because "DCIK ties LITE on lookups but wins decisively on high-stakes ambiguous decisions" is a
true, useful, and honest claim; "DCIK always wins" would be neither.

## Anti-Goodhart discipline (what makes it a demonstration, not a tuned number)

1. **Held-out.** Topics/gold used to *improve* DCIK are never used to *demonstrate* it. Demonstration
   runs use unseen topics + gold frozen before any arm exists.
2. **Independent arm generation.** Arms should not all be authored by the party that wants DCIK to
   win. Target: BASELINE + LITE generated by an independent agent/session; FULL run faithfully;
   ideally arm generation and scoring by different model families than the author. Where a run
   cannot achieve this, it is labelled with the confound (as the PoC was) and treated as directional.
3. **≥2 blind scorers**, blind to arm, scoring normalised outputs; report Cohen's κ.
4. **The eval must remain able to show failure.** A rubric/gold set that DCIK cannot lose against
   is worthless. Gold is derived from domain fundamentals and what an independent expert would
   demand — never reverse-engineered from DCIK's outputs.
5. **Report the honest verdict**, including "simplify" or "no measured value", if that is what the
   data says.

## Confirmatory bar

A confirmatory claim ("DCIK is demonstrably powerfully useful on high-stakes decisions") requires
**N ≥ 8 topics in the high-stakes class, ≥ 2 blind scorers, independent arm generation**, with FULL
beating LITE by ≥ the configured threshold on a majority, at acceptable κ. Anything less is
directional.
