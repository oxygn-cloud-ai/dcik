# Deepened perspectives (operation-forcing, independence-generating)

Each lens below is upgraded from a checklist of questions into a **protocol that produces an
independent estimate + a typed artifact + a decision-link**. The independence is the point: each
perspective reaches its OWN conclusion before any synthesis, so aggregation has genuinely divergent
inputs (Self-MoA logic), not one voice restated six ways.

---

## P0008 — Counterparty as Rational Adversary (deep)
**Operation:** adversarial valuation from the *other seat* + motive distribution.
1. Enumerate the acquirer's plausible motives; assign a probability to each (must sum to 1):
   defensive-block, acqui-hire, roadmap-gap/build-vs-buy, data/customer-grab, category-option.
2. For the top-2 motives, compute *their* walk-away price — the most a rational acquirer with that
   motive pays. Produce a number/range **independently of our $24M ask**.
3. **Independent estimate (required):** P(6× is an opening bid below their reservation price) with a
   one-line basis.
4. **Decision-link:** if that probability is high → the move is negotiate/press, not accept/decline.

## P0012 — Information Asymmetry (deep)
**Operation:** name the asymmetry, price it.
1. List what the acquirer plausibly knows that the founders don't (market timing, a competing
   product, churn signals, category consolidation) and what the founders know that the acquirer is
   likely underpricing.
2. Answer "why *now*?" with the single most likely acquirer-side reason.
3. **Independent estimate (required):** P(the acquirer is transacting now because the next 6–12
   months are *worse* than founders believe) + basis.
4. **Decision-link:** high → discount the "we can always raise later" option; low → optionality holds.

## P0013 — Challenge the Premise (deep)
**Operation:** premise extraction + inversion, whose-goal decomposition.
1. State the single load-bearing premise ("exit now is desirable") and whose objective it serves
   (founders / company / investors) — these can conflict.
2. Invert: model the world where the *right* move is to NOT run a process at all, and state the
   evidence that would put us in that world.
3. **Independent estimate (required):** which stakeholder's interest the current framing most serves,
   and the % of the decision that is really about founder liquidity vs company value.
4. **Decision-link:** if it's mostly founder-liquidity, the honest lever is a founder secondary, not
   a company sale.

## P0015 — Cognitive Bias, applied to the deciders (deep)
**Operation:** bias-source test (is the flagged bias actually moving the conclusion?).
1. Name the single bias most likely distorting THIS framing (anchoring on the $24M headline;
   loss-aversion post-raise; fatigue-driven FOMO).
2. Run the debiasing test: would the recommendation change if the headline number were 3× instead of
   6×, holding all else equal? If yes, the number is anchoring the analysis.
3. **Independent estimate (required):** P(the lean-toward-accept impulse is founder-state-driven
   rather than value-driven).
4. **Decision-link:** high → separate the personal-utility decision from the company-EV decision
   explicitly before recommending.

## P0018 — Incentive & Waterfall Analysis (deep)
**Operation:** cap-table waterfall + retention decomposition.
1. Compute the $24M waterfall: Series-A liquidation preference first, then option-pool/unvested
   treatment, then founder proceeds. State the founder *net* (not "55% × 24M").
2. Split the headline into cash-at-close vs escrow/earnout/retention using the base rate (20–30%
   contingent, earnouts realise ~50–70%). State the *effective* price.
3. **Independent estimate (required):** effective risk-adjusted founder take-home as a range, and
   who ($) the deal actually serves (founders / employees / investors).
4. **Decision-link:** if founder net is largely contingent/retention → "life-changing liquidity" is
   false and the accept case weakens sharply.

## P0002 — Financial / Reference-Class (deep)
**Operation:** outside-view base rates + calibrated EV of each path.
1. Build the reference class: Series-A B2B SaaS at ~$4M ARR / ~80% growth. Base rates —
   Series-A→B graduation ~9–25% within 2 yrs (2025 cohorts); distribution of outcomes (fail / flat /
   up-round / large exit).
2. Compute a rough EV of STAY (probability-weighted, dilution- and time-adjusted) vs the *effective*
   (post-waterfall, post-contingency) value of ACCEPT.
3. **Independent estimate (required):** P(staying independent yields a better risk-adjusted founder
   outcome than the effective accept value) — a calibrated number with confidence.
4. **Decision-link:** this probability, not the headline multiple, is the actual crux.

---

## Aggregation protocol (how deep-DCIK combines them)
- Collect the six **independent** probability estimates + artifacts.
- Where they bear on the same quantity (e.g. "is 6× a floor?"), aggregate (median) rather than let
  one voice dominate.
- Surface DISAGREEMENTS between perspectives explicitly (they are the signal), and resolve by
  evidence, not by the loudest lens.
- Output a decision that names the 1–2 probabilities that most move it, with calibrated confidence
  and an explicit "what would change the answer."
