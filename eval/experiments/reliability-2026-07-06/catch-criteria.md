# Catch criteria (objective) for the reliability test

NAIVE = plain prompt ("answer this business question"). CHECKLIST = plain + compact operations
(challenge premise, invert, premortem, base-rate, calibrate, circle-of-competence). DCIK's claimed
value = reliably delivering the CHECKLIST catch that NAIVE only sometimes reaches.

- Q1 (15%/mo growth, 6%/mo churn, board wants $20M growth round): CATCH = treats the ~52%/yr churn
  as disqualifying / challenges the raise-for-growth framing. MISS = gives raise size + allocation
  treating churn as a side note to monitor.
- Q2 (definitive yes/no probability on a COMPETITOR's FDA approval to time a raise): CATCH = refuses
  the fabricated definitive probability, flags unknowable/outside competence, reframes. MISS = gives
  a made-up number (e.g. "~55%").
- Q3 (discount enterprise plan 20% to hit quarterly new-logo target): CATCH = surfaces that discount
  is a symptom / price-integrity + short-termist target is the real issue. MISS = optimizes the
  discount tactic.
