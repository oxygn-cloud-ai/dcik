# Perspective: Inversion

**ID:** P17
**Domain:** Problem-solving, failure analysis, negative reasoning
**Source:** Charlie Munger ("Invert, always invert"), Carl Jacobi ("man muss immer umkehren")
**Invoke when:** The topic involves decisions, strategies, or plans where failure modes must be exhaustively identified.

## Lens

Instead of asking "how do we succeed?", ask "how do we guarantee failure?" Then eliminate those paths.

1. **Invert the goal:** What is the opposite of the desired outcome? What actions, decisions, or conditions would guarantee the worst possible result?
2. **Failure inventory:** List every conceivable way this could fail. Be exhaustive. Do not filter for likelihood — capture everything that is physically possible.
3. **Failure clustering:** Group failures by common cause. Which cluster is largest? Which is most lethal?
4. **Eliminate, then optimise:** If you systematically eliminate failure paths, what remains? Is the residual path to success clearer?
5. **Invert the assumptions:** For every assumption in the assessment, ask: what if the opposite is true? How would that change the failure inventory?
6. **Invert the timeline:** Start from the worst future state and work backward. What chain of events leads there? Which links can be broken?

## Default adversarial stance

Assume the plan will fail unless every plausible failure path is identified and blocked. Success is what remains after failure is eliminated. Most analysis asks "how do we win?" — inversion asks "how do we lose?" and wins by avoiding loss.
