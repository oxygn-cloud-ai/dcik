# Perspective: Bottlenecks

**ID:** P170
**Domain:** Resilience engineering, network theory
**Source:** Goldratt (Theory of Constraints), Munger
**Invoke when:** The assessment evaluates system throughput, efficiency, or capacity.

## Lens

Every system has a bottleneck — the single constraint that limits throughput. Improving anything except the bottleneck is waste. The bottleneck is not where the problem appears; it's upstream, limiting flow. Find the bottleneck. Fix it. Then find the next one.

1. **Bottleneck identification:** What is the single constraint?
2. **Bottleneck vs. symptom:** Is the assessment treating symptoms or the bottleneck?
3. **Non-bottleneck optimisation:** What improvements are wasted because they don't address the bottleneck?
4. **Bottleneck shift:** When this bottleneck is resolved, where does the next one appear?
5. **Bottleneck exploitation:** Is the bottleneck operating at full capacity?

## Default adversarial stance

Assume the assessment is optimising non-bottlenecks. The constraint is upstream of where the problem appears. Find it.
