# Perspective: Cynefin Framework

**ID:** P89
**Domain:** Decision-making, complexity, systems, strategy
**Source:** Dave Snowden (Cynefin framework)
**Invoke when:** The assessment applies the same analytical approach regardless of the type of problem — when complexity and context are ignored.

## Lens

Cynefin classifies problems into five domains, each requiring different analytical approaches. Applying the wrong domain's methods to a problem produces wrong answers. The domains: Clear (known knowns), Complicated (known unknowns — expert analysis needed), Complex (unknown unknowns — probe-sense-respond), Chaotic (act first to stabilise), and Confusion (don't know which domain you're in).

1. **Domain classification:** Which Cynefin domain does this problem belong to? Clear: cause and effect obvious. Complicated: cause and effect discoverable by experts. Complex: cause and effect only visible in retrospect. Chaotic: no discernible cause and effect.
2. **Method mismatch:** Is the assessment applying Clear-domain methods (best practice, standard procedures) to a Complex-domain problem (where emergent practice is needed)? Or vice versa?
3. **Expertise appropriateness:** In Complicated domains, expertise solves problems. In Complex domains, expertise is insufficient — experiments and probes are needed. In Chaotic domains, expertise is irrelevant — action and stabilisation come first.
4. **Probe-sense-respond:** For Complex problems: what probes (small, safe-to-fail experiments) could reveal the system's behaviour? What sensing mechanisms detect the results? How fast can the response adapt?
5. **Domain transition:** Is the problem transitioning between domains? A Complicated problem becoming Complex as uncertainty increases. A Clear problem becoming Chaotic due to external shock. Domain transitions are when most failures occur.
6. **Confusion detection:** Is the assessment in the Confusion domain — applying methods without knowing which domain applies? The first step out of Confusion is to classify the problem.

## Default adversarial stance

Assume the assessment is applying Clear-domain thinking to a Complex-domain problem. Assume expertise is being over-relied upon where probes would be more effective. The most dangerous analytical error is misclassifying the domain.
